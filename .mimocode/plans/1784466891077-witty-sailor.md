# Sage AI Detection System — Top-Tier Rebuild Plan

## Problem Summary

1. **No real accident detection** — COCO-SSD detects objects but can't determine if an accident is happening. Current collision detector requires 2+ "strong signals" that rarely fire together.
2. **Speed shows 0km/h** — Speed is pixel displacement between COCO-SSD detections (runs every 300ms). Between frames, object positions don't change → 0. No camera calibration exists.
3. **COCO-SSD is too slow** — 3 FPS detection can't capture rapid events (sudden stops, impacts).
4. **False positives in traffic** — Still firing on normal traffic despite all the threshold tuning.
5. **Classification still imperfect** — Even with COCO-SSD, the model can't distinguish Indian traffic well (auto-rickshaws, bikes in groups).

## Root Cause Analysis

The fundamental issue: **we're doing post-hoc analysis of sparse detections instead of real-time trajectory prediction.** We detect objects, then look for evidence AFTER something happens. By the time we see "hard brake" or "shape change," the accident already occurred 500ms ago.

The correct approach: **predict accidents BEFORE they happen** using trajectory extrapolation and time-to-collision (TTC) computation.

## Architecture: Three-Layer Detection System

```
Layer 1: YOLO26n ONNX (Object Detection) — 10+ FPS
   ↓ bounding boxes + class + confidence
Layer 2: Kalman Tracker (Object Tracking) — smooth trajectories
   ↓ tracked objects with predicted positions
Layer 3: Physics Engine (Accident Detection)
   ├── TTC (Time-to-Collision) computation
   ├── Trajectory anomaly detection
   ├── Speed estimation with camera calibration
   └── Post-impact confirmation
```

## Phase 2: Speed Estimation Fix

### Why speed shows 0km/h
Current code: `speed = Math.sqrt((cx2-cx1)² + (cy2-cy1)²)` — pixel displacement between consecutive detections. With COCO-SSD at 3 FPS, objects barely move between detections → ~0 pixels → 0 km/h.

### Fix: Multi-frame speed + camera calibration

**Step 1: Increase detection frequency**
- YOLO26n at 10+ FPS gives 3x more position updates
- Speed becomes meaningful: `speed = displacement / (1/10) = displacement * 10`

**Step 2: Kalman filter for smooth velocity estimation**
```typescript
// Instead of raw pixel displacement:
const rawSpeed = Math.sqrt(dx*dx + dy*dy);

// Use Kalman filter to smooth velocity:
const kalman = new KalmanFilter({ R: 0.01, Q: 0.1 });
const smoothVelocity = kalman.update(rawVelocity);
```

**Step 3: Camera calibration for real km/h**
- The `cameras` table has `calibration_data JSONB` — use it
- Auto-calibrate using vanishing point detection from lane markings
- Or manual: user clicks two points of known distance
- Convert: `realSpeed = pixelSpeed * (realDistanceMeters / pixelDistance) * fps * 3.6`

**Step 4: Auto-calibration heuristic**
```typescript
// Estimate pixelsPerMeter from video:
// - Road width ~3.5m per lane
// - Assume 2-3 lanes visible
// - Measure road width in pixels
// - pixelsPerMeter = roadWidthPixels / (lanes * 3.5)
```

## Phase 3: Trajectory-Based Accident Detection

### The Key Insight
Instead of detecting "evidence of collision after it happens," **predict collisions BEFORE they happen** using Time-to-Collision (TTC).

### TTC Algorithm
```typescript
function timeToCollision(obj1, obj2): number | null {
  // Relative position
  const rx = obj2.x - obj1.x;
  const ry = obj2.y - obj1.y;
  
  // Relative velocity
  const rvx = obj2.vx - obj1.vx;
  const rvy = obj2.vy - obj1.vy;
  
  // Closing speed (positive = approaching)
  const closingSpeed = -(rx * rvx + ry * rvy) / Math.sqrt(rx*rx + ry*ry);
  
  if (closingSpeed <= 0) return null; // Moving apart
  
  // Distance / closing speed = time to collision
  const dist = Math.sqrt(rx*rx + ry*ry);
  const combinedSize = (Math.sqrt(obj1.area) + Math.sqrt(obj2.area)) / 2;
  
  return (dist - combinedSize) / closingSpeed; // seconds
}
```

### Accident Detection Rules
1. **TTC < 1.0s** → WARNING (state: watching)
2. **TTC < 0.5s + deceleration** → CONFIRM (state: confirming)
3. **TTC < 0.2s OR sudden stop after TTC < 0.5s** → ALERT (state: alert)
4. **Post-impact**: bounding box overlap + both objects stopped = confirmed accident

### Trajectory Anomaly Detection
```typescript
// Detect abnormal trajectory changes
function detectTrajectoryAnomaly(obj: TrackedObject): boolean {
  if (obj.positions.length < 5) return false;
  
  // Fit linear trajectory to last 5 positions
  const recentPositions = obj.positions.slice(-5);
  const predictedNext = linearPredict(recentPositions);
  const actualNext = obj.positions[obj.positions.length - 1];
  
  // Large deviation from predicted path = anomaly
  const deviation = distance(predictedNext, actualNext);
  const expectedDeviation = obj.speed * 0.3; // 30% tolerance
  
  return deviation > expectedDeviation;
}
```
