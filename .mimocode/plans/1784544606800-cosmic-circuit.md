**Detection Algorithm — Vehicle Collision:**

```
Score = Σ(signal_i × weight_i) where signals are:

1. OVERLAP (weight: 0.30)
   - IoU between bounding boxes
   - Score: min(IoU × 5, 1.0) — 20% IoU = full score

2. DECELERATION (weight: 0.25)  
   - Both vehicles must show speed drop > threshold
   - Score: (a_decel + b_decel) / 2
   - Per vehicle: speed_history[0..2] avg vs speed_history[3..5] avg
   - Drop > 40% = decelerating

3. PROXIMITY (weight: 0.20)
   - Distance between centers < combined_radius × 0.7
   - Score: 1.0 if touching, 0.5 if close, 0.0 if far

4. TRAJECTORY CONVERGENCE (weight: 0.15)
   - Using trajectory-predictor: will paths intersect within 10 frames?
   - Score: 1.0 if collision predicted, 0.5 if close approach, 0.0 if diverging

5. ENERGY TRANSFER (weight: 0.10)
   - Using energy-analyzer: did one vehicle lose KE while other is nearby?
   - Score: 1.0 if energy transfer detected, 0.0 otherwise
```

**Mode-Specific Behavior:**

| Parameter | Isolated | Traffic | Marketplace |
|-----------|----------|---------|-------------|
| Min score threshold | 0.40 | 0.65 | 0.55 |
| Min signals required | 2 | 3 | 3 |
| Overlap required | No | No | No |
| Deceleration required | No | Yes (both) | No |
| Passing filter | Basic | Strict | N/A |
| Confirmation frames | 3 | 5 | 5 |
| Cooldown | 3s | 8s | 8s |

**Passing Vehicle Filter (Traffic Mode — CRITICAL for false positive prevention):**

```
isPassing(a, b) → boolean:
  // Both vehicles moving at reasonable speed
  if (a.speed < 0.8 || b.speed < 0.8) return false
  // Same direction (within 45 degrees)
  headingDiff = angleDiff(a.heading, b.heading)
  if (headingDiff < π/4) return true
  // Both maintaining speed (no deceleration)
  if (!hasDecelerated(a) && !hasDecelerated(b)) return true
  return false
```

## Phase 2: Worker Rebuild

### 2.1 `lib/worker/detection-worker.ts` — MAJOR REWRITE

This is the most critical file. Currently has its own inline collision detection that ignores the TTC engine. Rebuild to use the unified pipeline.

**New Architecture:**

```
Worker receives FRAME → 
  1. ONNX inference (onnx-engine.ts)
  2. Kalman tracking (kalman-tracker.ts) 
  3. Frame memory update (frame-memory.ts)
  4. UNIFIED detection pipeline:
     a. detectAccidents(entities, envMode) — from ttc-engine.ts
     b. energy analysis — from energy-analyzer.ts
     c. trajectory analysis — from trajectory-predictor.ts
  5. Temporal confirmation (sustained evidence over N frames)
  6. False positive filtering (mode-specific)
  7. Serialize and post results
```

**Key Changes:**

1. **Import and use ttc-engine.ts** instead of inline detectCollisions()
2. **Import and use energy-analyzer.ts** as supplementary signal
3. **Import and use trajectory-predictor.ts** as supplementary signal
4. **Implement mode-aware behavior** — SET_MODE handler actually changes envMode variable
5. **Add temporal confirmation buffer** — evidence must persist for N frames before alerting
6. **Add false positive blacklist** — patterns known to be false positives
7. **Fix the demo mode** — keep it but make it produce realistic collision scenarios

**Temporal Confirmation System:**

```typescript
interface ConfirmationBuffer {
  evidenceKey: string;  // e.g. "collision:3,5" (type + object IDs)
  firstSeen: number;    // frame number
  lastSeen: number;     // frame number  
  maxConfidence: number;
  signalHistory: number[];  // confidence per frame
}

// In worker:
const confirmBuffer = new Map<string, ConfirmationBuffer>();

function checkConfirmation(evidence: Evidence, frame: number): boolean {
  const key = `${evidence.type}:${[...evidence.objects].sort().join(',')}`;
  const existing = confirmBuffer.get(key);
  
  const minFrames = envMode === 'isolated' ? 3 : 5;
  
  if (!existing) {
    confirmBuffer.set(key, {
      evidenceKey: key,
      firstSeen: frame,
      lastSeen: frame,
      maxConfidence: evidence.confidence,
      signalHistory: [evidence.confidence],
    });
    return false;
  }
  
  existing.lastSeen = frame;
  existing.maxConfidence = Math.max(existing.maxConfidence, evidence.confidence);
  existing.signalHistory.push(evidence.confidence);
  if (existing.signalHistory.length > 10) existing.signalHistory.shift();
  
  // Must have enough frames
  if (existing.lastSeen - existing.firstSeen < minFrames) return false;
  
  // Confidence must be sustained (avg of last 3 frames > threshold)
  const recent = existing.signalHistory.slice(-3);
  const avgConf = recent.reduce((a,b) => a+b, 0) / recent.length;
  
  return avgConf > getModeThreshold(envMode);
}

function getModeThreshold(mode: EnvMode): number {
  switch (mode) {
    case 'isolated': return 0.40;
    case 'traffic': return 0.65;
    case 'marketplace': return 0.55;
  }
}
```

## Phase 4: Confidence Meter & Alert System

### 4.1 Confidence Scoring — Unified Formula

All detection paths now use the same formula:

```
Final Confidence = Base Score × Temporal Multiplier × Mode Multiplier × Quality Multiplier

Where:
- Base Score = Σ(signal_i × weight_i) from 5 signals
- Temporal Multiplier = min(1.0, sustained_frames / required_frames)
- Mode Multiplier = { isolated: 1.0, traffic: 0.85, marketplace: 0.9 }
  (traffic/marketplace get lower multiplier because more false positives)
- Quality Multiplier = based on image quality (low light = 0.9, good light = 1.0)
```

### 4.2 Severity Classification

```
severity = 
  confidence > 0.85 && mode === 'isolated' → 'critical'
  confidence > 0.75 → 'critical'  
  confidence > 0.60 → 'major'
  confidence > 0.45 → 'minor'
  else → 'suspicious'
```

## Key Detection Rules Per Mode

### Isolated Mode (Empty road, few objects)
- **Easiest to detect**: any collision is likely real
- Low threshold (0.40), fewer confirmation frames (3)
- Vehicles + pedestrians: low traffic = any accident is significant
- Scene context: few objects → high confidence in detections

### Traffic Mode (Busy road, many vehicles)
- **Hardest mode**: passing vehicles constantly trigger overlap
- High threshold (0.65), more confirmation frames (5)
- **Mandatory deceleration check**: BOTH vehicles must decelerate
- **Strict passing filter**: same-direction fast vehicles = reject
- **Mutual proximity requirement**: both vehicles must be near each other AND decelerating
- **History check**: vehicles must have been approaching before the "collision"

### Marketplace (Pedestrians, few vehicles)
- Vehicle collisions rare but serious
- Pedestrian falls more likely
- Crowd anomalies suppressed (crowds are normal)
- Vehicle detection threshold higher (fewer vehicles = more confident when detected)
- Person-related events: lower threshold for falls, higher for collisions

## False Positive Prevention Summary

| FP Type | Prevention Method |
|---------|-------------------|
| Passing vehicles (traffic) | Heading similarity + no deceleration = reject |
| Parked cars overlapping | Long-term position stability check (>30 frames) |
| Camera shake | Scene-wide movement detection = suppress all alerts |
| Single-frame blip | Temporal confirmation (3-5 frames sustained) |
| Low confidence blips | Mode-specific minimum thresholds |
| Crowd movement | Marketplace mode suppresses crowd alerts |
| Person bending down | Aspect ratio change must be sustained + speed drop |
| Shadow/ghost detection | Minimum age requirement (3+ frames) |
